# AGENTS.md

本文件记录 `paper_rag` 项目的开发规范。所有 Agent 或协作者在修改本仓库前，应先阅读并遵守本文约定。

## 1. 项目环境
- 本项目使用 conda 环境 `paper_rag`。
- 本项目用 D:\ai_study_app\anaconda3\envs\paper_rag\python.exe 来执行所有 Python / pytest / 脚本命令


## 2. 注释与文档化要求
- 所有说明和注释用中文
- 所有文件都必须有文件级说明。
- 所有公开接口都必须有注释。
- 所有枚举都必须有注释，说明枚举的业务含义。
- 所有类型都必须有注释。
- 类型中的每个字段都必须有注释或描述，说明字段含义、用途和必要约束。
- 注释应解释“为什么存在、代表什么、如何使用”，不要写无意义的机械重复，例如“设置变量 value”。

## 3.扩展性原则(写代码与设计时遵循)
本项目进行代码编写时应该考虑扩展性和可维护性。
具体把握以下区分:
- 在"改起来贵"的地方必须预留扩展性:数据模型、数据库结构、后端架构、模块分层、API 接口设计。这些事后重构成本极高,从一开始就要按"将来会有大量真实用户/数据"来设计。
- 在"改起来便宜"的地方按当前阶段需要来做,不必过度设计:具体的界面形态、视觉样式、交互细节。这些改动成本低,且不同阶段本就该有不同形态,不要为想象中的未来形态牺牲当前的体验和清晰度。
- 判断准则:预留扩展性,是为了避免"未来不得不推倒重写";不是为了"现在就把每个细节都做成未来的样子"。
